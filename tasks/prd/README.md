# PRDs

Larger plans live here. Each file is a self-contained PRD with
problem statement, user stories, implementation decisions, testing
decisions, and out-of-scope notes. Status starts at `needs-triage`
and moves through your normal triage flow before
`/to-issues` decomposes it into vertical-slice tickets in
`tasks/issues/`.
